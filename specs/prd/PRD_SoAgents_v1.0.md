## 1. 产品愿景与定位

### 1.1 产品愿景

**让每个人都能拥有一个真正能干活的 AI Agent**

2026 年是智能丰裕的元年，AI 已经让开发者成为 10x-100x 生产力的人。SoAgents 的使命是将这种力量民主化——无论你是学生、内容创作者、教育工作者、产品经理，还是任何「想要做些什么的人」，都能通过直观的图形界面，让 AI Agent 成为你的思维放大器。

### 1.2 产品定位

SoAgents 是基于 Claude Agent SDK 的**开源桌面端通用 Agent 产品**。

**核心差异化**：

| 维度 | SoAgents | Claude Code CLI | 其他 Agent 产品 |
|------|----------|-----------------|-----------------|
| 使用门槛 | GUI 零门槛 | 需要命令行 | 多需要编程 |
| 并行能力 | 多 Tab 真并行 | 单会话 | 单会话 |
| 模型灵活性 | 多供应商可选 | 仅 Anthropic | 固定供应商 |
| 数据隐私 | 本地存储 | 本地 | 多为云端 |
| 开源 | Apache 2.0 | 闭源 | 多为闭源 |

### 1.3 产品信息

## 2. 目标用户

### 2.1 主要用户群

| 用户类型 | 典型场景 | 核心需求 |
|----------|----------|----------|
| **内容创作者** | 写作、剪辑脚本、社媒运营 | 快速生成、批量处理、风格一致 |
| **学生/研究者** | 论文整理、代码学习、资料分析 | 学习辅助、知识整理、代码解释 |
| **产品经理** | PRD 撰写、竞品分析、数据整理 | 文档生成、信息提取、格式转换 |
| **独立开发者** | 原型开发、代码生成、调试 | 快速迭代、多项目并行、MCP 扩展 |
| **各行业专家** | 领域知识应用、自动化任务 | 专业 Prompt、工作流定制 |

### 2.2 用户特征

**技术背景**：无需编程经验，但具备基本计算机操作能力

**核心诉求**：
- ✅ 不想学命令行，但想用 AI Agent 的强大能力
- ✅ 需要同时处理多个项目/任务
- ✅ 关注数据隐私，不希望数据上传云端
- ✅ 希望灵活选择模型供应商（成本/性能平衡）

### 2.3 非目标用户

- ❌ 纯命令行偏好的重度开发者（Claude Code CLI 更适合）
- ❌ 需要企业级协作功能的团队（暂不支持）
- ❌ 需要移动端使用的用户（仅桌面端）

## 4. 技术架构

### 4.1 核心概念

在阅读架构之前，先理解以下核心概念：

| 概念 | 定义 | 类比 |
|------|------|------|
| **Tab（标签页）** | 一个独立的聊天窗口，可关联特定项目目录 | 类似浏览器的标签页，每个 Tab 是一个独立的「对话空间」 |
| **Sidecar（边车进程）** | 为 Tab 提供 AI 能力的后端进程 | 类似每个 Tab 配备了一个专属的「AI 助手引擎」 |
| **Session（会话）** | Tab 内的对话历史和上下文 | 一次完整的对话记录，包含所有消息和状态 |
| **工作区** | Tab 关联的项目目录 | AI 可以读写该目录下的文件 |

**多 Tab 工作示意**：

```
┌─────────────────────────────────────────────────────────────────┐
│  [Tab 1: 前端项目]  [Tab 2: 后端项目]  [Tab 3: 文档整理]  [+]   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Tab 1 (当前)                    │  Tab 2 (后台运行)            │
│  工作区: ~/projects/frontend     │  工作区: ~/projects/backend  │
│  状态: 等待输入                   │  状态: 正在执行代码重构...    │
│                                  │                              │
│  你可以在 Tab 1 继续对话          │  Tab 2 的任务在独立进程中     │
│  不会被 Tab 2 的任务阻塞          │  并行执行，互不干扰           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**为什么需要独立进程？**

| 场景 | 单进程（传统方案） | 多进程（SoAgents） |
|------|-------------------|-------------------|
| Tab 1 执行长任务 | Tab 2 必须等待 | Tab 2 正常使用 |
| Tab 1 崩溃 | 整个应用崩溃 | 仅 Tab 1 受影响 |
| 不同项目配置 | 需要手动切换 | 每个 Tab 独立配置 |

```
┌─────────────────────────────────────────────────────────────────┐
│                       Tauri Desktop App                          │
├──────────────────────────────────────────────────────────────────┤
│                         React Frontend                           │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │

│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘              │

├─────────┼────────────────┼────────────────┼──────────────────────┤
```

### 4.3 技术栈

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **桌面框架** | Tauri v2 (Rust) | 体积小、性能高、安全性好 |
| **前端** | React 19 + TypeScript + Vite | 生态成熟、类型安全、开发效率高 |
| **样式** | TailwindCSS | 原子化 CSS、快速迭代 |
| **后端** | Bun + Claude Agent SDK | 启动快、内置运行时、官方 SDK |
| **通信** | Rust HTTP/SSE Proxy | 绕过 WebView CORS、低延迟 |
| **存储** | 本地文件系统 (JSONL) | 隐私保护、无需数据库 |

### 4.4 核心设计原则

| 原则 | 说明 |
|------|------|
| **Tab 隔离** | 每个 Tab 独立 Sidecar 进程，崩溃隔离、真并行 |
| **Rust 代理** | 所有网络请求经 Rust 层，安全可控 |
| **零外部依赖** | 内置 Bun 运行时，用户无需安装 Node.js |
| **Session 保持** | 切换模型/供应商不丢失对话上下文 |

### 4.5 数据流向

```
用户输入 → React Frontend → Tauri IPC → Rust Proxy → Bun Sidecar
                                                          ↓
                                                   Claude Agent SDK
                                                          ↓
                                                   AI 模型 API
                                                          ↓
SSE 实时推送 ← Rust SSE Proxy ← Bun Sidecar ← 流式响应
      ↓
React 渲染更新
```

### 6.2 竞争优势

**相比 Claude Code CLI**：
- ✅ 图形界面，无需命令行经验
- ✅ 多 Tab 真并行，同时处理多个项目
- ✅ 支持多模型供应商，成本可控
- ✅ 开源免费

**相比 Cursor/Copilot**：
- ✅ 不依赖特定 IDE，独立运行
- ✅ 完整的 Agent 能力（不仅是代码补全）
- ✅ MCP 工具扩展生态
- ✅ 数据完全本地化

**相比 ChatGPT Desktop**：
- ✅ 真正的 Agent 能力（可执行文件操作）
- ✅ 多模型支持（不限于 OpenAI）
- ✅ 专为生产力设计（工作区、多 Tab）

### 6.3 竞争劣势与应对

| 劣势 | 应对策略 |
|------|----------|
| 品牌知名度低 | 开源社区运营、内容营销 |
| 功能深度不如 Cursor | 聚焦通用场景，而非专业 IDE |
| 无移动端 | 专注桌面端体验，移动端待规划 |
| 无团队协作 | 定位个人用户，企业版待规划 |

## 9. 风险评估

### 9.1 技术风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| Claude SDK 重大变更 | 中 | 高 | 抽象适配层，跟进官方更新 |
| Tauri v2 稳定性问题 | 低 | 中 | 紧跟社区，及时升级 |
| 多进程内存压力 | 中 | 中 | 进程池管理，限制最大 Tab 数 |
| MCP 协议变更 | 低 | 中 | 模块化 MCP 层，易于适配 |

### 9.2 产品风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 用户增长缓慢 | 中 | 高 | 内容营销、社区运营、口碑传播 |
| 功能同质化 | 中 | 中 | 聚焦差异化（多 Tab、多供应商、开源） |
| 用户需求分散 | 高 | 中 | 明确目标用户，拒绝 feature creep |
| 竞品快速跟进 | 中 | 中 | 快速迭代，保持技术领先 |

### 9.3 运营风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 开源社区不活跃 | 中 | 中 | 积极响应 Issue/PR，定期发布 |
| 文档维护滞后 | 高 | 低 | 版本发布流程强制更新文档 |
| API 供应商限制 | 低 | 高 | 多供应商支持，不依赖单一供应商 |

### 9.4 法律风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 开源协议合规 | 低 | 中 | 严格遵循 Apache 2.0，依赖审计 |
| 商标问题 | 低 | 中 | 注册商标，品牌保护 |
| 数据合规（GDPR 等） | 低 | 中 | 数据本地化，无云端存储 |
