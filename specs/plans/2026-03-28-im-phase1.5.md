# IM Phase 1.5 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Polish Phase 1 IM integration with runtime status display, proxy URL support, and buffer replay.

**Architecture:** Three independent tasks touching Rust + React. Each is self-contained.

**Tech Stack:** Rust (Tauri), React + TypeScript, TailwindCSS

**Note:** Phase 1.5 PRD items 7.2 (streaming) and 7.5 (backoff) are already fully implemented. This plan covers the remaining gaps.

### Task 1: Runtime Status Info in ChannelConfigPanel

**Files:**
- Modify: `src/renderer/components/ImAgentSettings/channels/ChannelConfigPanel.tsx`

When a channel is online, show uptime, active session count, last message time, error message, and buffered message count. Data is already returned by `getAllChannelsStatus()` → `ImBotStatus`.

- [ ] **Step 1: Add status info section below header when online**

- Active sessions count
- Last message time (relative, e.g. "5 minutes ago")
- Error message (if any, in red)
- Buffered messages count (if > 0, in warning color)
